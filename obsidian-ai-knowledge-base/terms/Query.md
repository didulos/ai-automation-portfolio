# Query

## 정의
사용자 또는 시스템이 데이터베이스, 검색 엔진, 또는 모델에서 특정 정보를 찾기 위해 제시하는 요청이다. [[Information-Retrieval]](정보 검색) 시스템에서 쿼리는 사용자의 의도(intent)를 표현하는 입력값으로, 관련성 높은 결과물을 반환받기 위한 핵심 요소다.

## 왜 중요한가
[[AI-Terms/01-Terms/RAG]](검색증강생성)나 [[AI-Terms/01-Terms/NLP]](자연어처리) 기반 시스템에서 쿼리의 품질이 직접 결과물의 정확도(precision)와 재현율(recall)을 결정한다. 실무에서 쿼리를 최적화하면 검색 성능과 사용자 만족도를 크게 향상시킬 수 있다.

## 관련 개념
- **상위 개념**: [[Information-Retrieval]](정보 검색)
- **하위 개념**:
  - [[Semantic-Search]](의미론적 검색) — 단순 키워드 매칭이 아닌 의미 기반 검색
  - [[Ranking]](순위 매김) — 검색 결과를 관련성순으로 정렬
- **연관 개념**:
  - [[Embedding]](임베딩) — 쿼리를 벡터(vector)로 변환하여 유사도 계산
  - [[Vector-Database]](벡터 데이터베이스) — 쿼리 벡터와 문서 벡터 간 유사성 검색

## 비유로 이해하기
쿼리는 도서관에서 사서에게 묻는 "질문"과 같다. 질문이 명확할수록 원하는 책을 빨리 찾을 수 있으며, 모호한 질문은 관계없는 책들을 받게 된다.

## 실제 사용 예시
- **검색 엔진**: 사용자가 "최신 머신러닝 논문"이라고 입력한 쿼리가 [[Semantic-Search]] 모델을 통해 [[Embedding]]으로 변환되어 [[Vector-Database]]에서 관련 논문들을 검색한다.
- **RAG 시스템**: LLM 기반 챗봇에서 사용자 질문(쿼리)을 [[Embedding]]한 후, 외부 지식베이스에서 가장 유사한 문서를 [[Retrieval]](검색)하여 응답 생성에 활용한다.
