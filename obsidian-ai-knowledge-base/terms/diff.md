# diff

## 정의
두 파일이나 텍스트의 차이점을 비교하여 어떤 부분이 추가, 삭제, 변경되었는지 보여주는 도구 또는 연산이다. [[Version-Control]](버전 관리) 시스템에서 코드 변경 사항을 추적하고 가시화하는 데 핵심적인 역할을 한다.

## 왜 중요한가
머신러닝 프로젝트에서 코드, 데이터, 모델 체크포인트의 변경 사항을 정확히 파악해야 실험의 재현성(reproducibility)을 보장할 수 있다. [[Git]](깃) 같은 현대적 [[Version-Control]](버전 관리) 시스템의 기본이 되어 팀 협업 효율성을 크게 향상시킨다.

## 관련 개념
- **상위 개념**: [[Version-Control]](버전 관리)
- **하위 개념**: 
  - [[Unified-Diff]](유니파이드 diff) - 표준 diff 포맷
  - [[Patch]](패치) - diff 결과를 적용하는 작업
- **연관 개념**: 
  - [[Git]](깃) - diff 기능을 포함한 분산 버전 관리 시스템
  - [[Merge]](병합) - 서로 다른 변경사항을 통합할 때 diff 정보 활용

## 비유로 이해하기
문서 두 개를 나란히 놓고 빨간색으로 삭제된 부분, 초록색으로 추가된 부분을 표시하는 것처럼, diff는 원본과 수정본의 정확한 차이를 색으로 구분하여 보여준다.

## 실제 사용 예시
```bash
git diff main feature-branch  # main 브랜치와 feature-branch 사이의 모든 변경사항 표시
git diff HEAD~1 HEAD -- model.py  # 최근 커밋에서 model.py 파일의 변경사항만 확인
```
