# Prompt-Engineering

## 정의
프롬프트 엔지니어링(Prompt Engineering)은 [[LLM]](대규모 언어 모델)에 입력하는 텍스트 지시문을 체계적으로 설계하고 최적화하여 원하는 품질의 출력을 얻는 기술이다. 효과적인 질문 구조, 컨텍스트 제공, 예시(few-shot) 등을 통해 모델의 성능을 극대화한다.

## 왜 중요한가
동일한 모델이라도 프롬프트 품질에 따라 결과의 정확도와 유용성이 크게 달라지므로, 실무에서 최소한의 비용으로 최대의 성능을 끌어내기 위한 필수 기술이다. 특히 [[Fine-Tuning]](파인튜닝) 없이도 모델을 효과적으로 활용할 수 있게 해준다.

## 관련 개념
- **상위 개념**: [[Natural-Language-Processing]](자연어 처리)
- **하위 개념**: 
  - [[Few-Shot-Prompting]](소수 샘플 학습 프롬프팅): 소수의 예시를 제공하여 패턴을 학습하도록 유도
  - [[Chain-of-Thought]](사슬 추론): 단계별 논리 과정을 명시하도록 지시
- **연관 개념**: 
  - [[AI-Terms/01-Terms/RAG]](검색 증강 생성): 프롬프트에 외부 정보를 동적으로 추가
  - [[Prompt-Injection]](프롬프트 주입): 악의적 입력으로 모델을 조종하려는 공격 기법

## 비유로 이해하기
마치 요리사에게 요리를 주문할 때 단순히 "맛있는 음식"이라고 하는 것보다 "신선한 재료로 매콤하게 양념한 한우 볶음"이라고 구체적으로 지시하는 것처럼, [[LLM]](대규모 언어 모델)에 명확하고 구체적인 지시를 주면 훨씬 좋은 결과를 얻을 수 있다.

## 실제 사용 예시
**효과적이지 않은 프롬프트**: "파이썬 코드를 작성해줘"
**효과적인 프롬프트**: "사용자의 입력 리스트에서 중복 요소를 제거하는 파이썬 함수를 작성해줘. 시간 복잡도는 O(n)이어야 하고, 원본 순서는 유지해야 해."
