# REST-API

## 정의
[[REST-API]](REST API)는 [[HTTP]](하이퍼텍스트 전송 프로토콜) 프로토콜을 기반으로 웹 서비스 간 데이터를 주고받기 위한 아키텍처 스타일이다. 자원(resource)을 [[URI]](균일 자원 식별자)로 표현하고, [[HTTP-Methods]](HTTP 메서드)를 통해 해당 자원에 대한 조작(생성, 읽기, 수정, 삭제)을 수행한다.

## 왜 중요한가
현대 웹 애플리케이션과 마이크로서비스 아키텍처에서 표준적인 통신 방식이며, 간단하고 확장성 있는 인터페이스를 제공하기 때문에 실무에서 필수적이다. 다양한 클라이언트(모바일, 웹, IoT)와의 상호운용성을 보장한다.

## 관련 개념
- **상위 개념**: [[Web-Service]](웹 서비스)
- **하위 개념**:
  - [[HTTP-Methods]](HTTP 메서드): GET, POST, PUT, DELETE 등
  - [[JSON]](제이슨): API 응답 및 요청 데이터 포맷
- **연관 개념**:
  - [[Endpoint]](엔드포인트): API의 특정 기능을 담당하는 [[URI]] 경로
  - [[Authentication]](인증): [[API-Key]], [[OAuth]], [[JWT]] 등을 통한 접근 제어

## 비유로 이해하기
식당의 메뉴판과 주문 시스템에 비유할 수 있다. 메뉴판의 각 음식이 자원([[Resource]])이고, "주문한다", "수정한다", "취소한다" 같은 행동이 [[HTTP-Methods]]이며, 웨이터와 손님의 상호작용이 API 통신이다.

## 실제 사용 예시
```
GET /api/users/123 → ID 123인 사용자 정보 조회
POST /api/users → 새로운 사용자 생성
PUT /api/users/123 → ID 123인 사용자 정보 수정
DELETE /api/users/123 → ID 123인 사용자 삭제
```
