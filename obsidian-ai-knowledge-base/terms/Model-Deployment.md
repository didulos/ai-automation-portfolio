# Model-Deployment

## 정의
학습이 완료된 [[AI-Terms/01-Terms/Machine-Learning]](머신러닝) 모델을 프로덕션 환경(production environment)에 배포하여 실제 사용자나 시스템이 접근할 수 있도록 만드는 과정이다. [[Model-Deployment]](모델 배포)는 단순히 모델을 서버에 올리는 것을 넘어 모니터링, 유지보수, 버전 관리까지 포함한 전체 생명주기 관리를 의미한다.

## 왜 중요한가
아무리 뛰어난 모델도 실제 운영 환경에서 안정적으로 작동하지 않으면 가치가 없다. [[Model-Deployment]](모델 배포)는 개발 단계의 모델을 비즈니스 가치를 창출하는 자산으로 전환하는 필수 단계이며, [[Model-Drift]](모델 드리프트) 감지 및 성능 저하 대응 능력을 제공한다.

## 관련 개념

**상위 개념:**
- [[AI-Terms/01-Terms/MLOps]](머신러닝 운영)

**하위 개념:**
- [[Model-Serving]](모델 서빙): 배포된 모델을 API나 배치(batch) 형태로 예측 요청에 응하게 하는 기술
- [[Model-Versioning]](모델 버전 관리): 여러 버전의 모델을 관리하고 롤백(rollback)할 수 있도록 하는 시스템

**연관 개념:**
- [[CI/CD]](지속적 통합/배포): 모델 코드와 데이터 파이프라인을 자동으로 테스트하고 배포하는 프레임워크
- [[Model-Monitoring]](모델 모니터링): 배포된 모델의 성능과 데이터 품질을 지속적으로 관찰하는 활동

## 비유로 이해하기
[[Model-Deployment]](모델 배포)는 건축가가 설계한 건물 설계도(학습된 모델)를 실제로 지어서(배포해서) 사람들이 사용할 수 있도록 만드는 과정과 같다. 설계도가 아무리 훌륭해도 지어지지 않으면 쓸모없고, 준공 후에도 정기적인 점검과 유지보수가 필요하다.

## 실제 사용 예시
추천 시스템 모델을 [[Docker]](도커) 컨테이너로 패키징하여 [[Kubernetes]](쿠버네티스) 클러스터에 배포하고, [[API]](API) 서버로 실시간 예측을 제공하며 매일 [[Model-Monitoring]](모델 모니터링)을 통해 정확도 저하 여부를 확인한다.
