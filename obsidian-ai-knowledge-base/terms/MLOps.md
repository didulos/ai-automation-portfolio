# MLOps

## 정의

[[MLOps]](머신러닝옵스)는 머신러닝 모델을 개발(development)에서 운영(operations)까지 전체 생명주기에 걸쳐 관리하고 자동화하는 방법론이다. [[DevOps]](데브옵스)의 원칙을 머신러닝에 적용하여 모델의 배포(deployment), 모니터링(monitoring), 재훈련(retraining)을 체계적으로 수행한다.

## 왜 중요한가

프로덕션 환경에서 머신러닝 모델은 시간이 지남에 따라 성능이 저하되고(model drift), 새로운 데이터에 대응해야 하기 때문에 지속적인 관리와 자동화가 필수적이다. [[MLOps]]를 통해 개발팀과 운영팀 간의 협업을 강화하고 모델 배포 속도와 안정성을 동시에 확보할 수 있다.

## 관련 개념

- **상위 개념**: [[DevOps]](데브옵스)
- **하위 개념**: 
  - [[CI/CD]](지속적 통합/배포) - 모델 코드와 학습 파이프라인의 자동 테스트 및 배포
  - [[Model-Monitoring]](모델 모니터링) - 프로덕션 모델의 성능 지표 추적 및 이상 감지
- **연관 개념**: 
  - [[Data-Pipeline]](데이터 파이프라인) - 데이터 수집부터 전처리까지의 자동화된 흐름
  - [[Model-Registry]](모델 레지스트리) - 학습된 모델의 버전 관리 및 메타데이터 저장소
  - [[Feature-Store]](피처 스토어) - 재사용 가능한 특성(feature)의 중앙 집중식 관리 시스템

## 비유로 이해하기

[[MLOps]]는 소프트웨어 개발의 [[CI/CD]] 파이프라인처럼, 머신러닝 모델도 "개발 → 테스트 → 배포 → 운영 → 피드백"의 자동화된 순환 과정을 구축하는 것이다. 마치 공장의 조립 라인처럼 모든 단계가 체계적으로 흘러가고, 문제가 생기면 자동으로 감지되어 대응된다.

## 실제 사용 예시

팀이 새로운 추천(recommendation) 모델을 개발하면 [[MLOps]] 파이프라인을 통해 자동으로 검증(validation)되고, 성능이 기준을 충족하면 실시간 서빙 환경에 배포되며, 이후 지속적으로 성능 메트릭(metric)을 모니터링하다가 정확도(accuracy)가 떨어지면 자동으로 재훈련을 트리거한다.
