# Git

## 정의
[[Git]](깃)은 파일의 변경 이력을 추적하고 관리하는 분산형 버전 관리 시스템(distributed version control system)이다. 개발자들이 코드 변경사항을 기록하고, 이전 버전으로 돌아가며, 여러 브랜치에서 병렬로 작업할 수 있게 해준다.

## 왜 중요한가
협업 개발 환경에서 코드 변경을 추적하고 충돌을 관리하는 필수 도구이며, 모든 개발자가 반드시 숙련해야 하는 기본 기술이다. 또한 코드 품질 관리와 프로젝트 히스토리 유지의 기반이 된다.

## 관련 개념
- **상위 개념**: [[Version-Control]](버전 관리)
- **하위 개념**: 
  - [[Commit]](커밋): 변경사항을 저장소에 기록하는 단위
  - [[Branch]](브랜치): 독립적인 작업 흐름을 만드는 기능
- **연관 개념**: 
  - [[GitHub]](깃허브): Git 저장소를 호스팅하는 클라우드 플랫폼
  - [[Merge]](병합): 여러 브랜치의 변경사항을 통합하는 작업

## 비유로 이해하기
Git은 구글 Docs의 '버전 기록' 기능처럼, 문서의 모든 변경 과정을 기록하되, 훨씬 더 강력하게 여러 사람이 동시에 다른 버전에서 작업하고 나중에 합칠 수 있는 시스템이다.

## 실제 사용 예시
```
git commit -m "사용자 인증 기능 추가"
git push origin feature/login
```
