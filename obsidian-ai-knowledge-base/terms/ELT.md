# ELT

## 정의
[[ELT]](추출-적재-변환)는 데이터 소스에서 원본 데이터를 추출(Extract)하여 데이터 웨어하우스에 먼저 적재(Load)한 후, 그곳에서 필요한 형태로 변환(Transform)하는 데이터 파이프라인 패러다임이다. [[ETL]](추출-변환-적재)과 달리 변환 전에 데이터를 먼저 저장하므로 대용량 데이터 처리에 유리하다.

## 왜 중요한가
클라우드 환경과 분산 처리 기술의 발전으로 [[ELT]]는 대규모 데이터 처리에서 [[ETL]]보다 더 빠르고 유연한 방식이 되었으며, 현대적 데이터 아키텍처의 표준이다. 데이터 레이크(data lake)와 클라우드 웨어하우스 환경에서 필수적인 프로세스이다.

## 비유로 이해하기
물류 창고에 먼저 원자재를 모두 반입(적재)한 후, 필요할 때마다 그곳에서 가공(변환)하는 방식과 같다. [[ETL]]은 공장에서 먼저 완성품으로 만들어(변환) 창고에 보관하는 방식과 대조된다.

## 실제 사용 예시
AWS Glue, Google Cloud Dataflow, Apache Spark를 사용하여 S3 데이터 레이크에 원본 CSV를 직접 적재한 후, [[SQL]] 기반 변환 작업을 수행한다. Snowflake와 같은 클라우드 웨어하우스에서 [[ELT]] 방식으로 구조화되지 않은 데이터를 먼저 수집하고, 이후 비즈니스 요구에 맞게 변환하는 유연성을 확보한다.
