# Self-Consistency

## 정의

[[Self-Consistency]](셀프 컨시스턴시)는 동일한 문제에 대해 [[LLM]](대형언어모델)이 여러 개의 다른 추론 경로(reasoning path)를 생성한 후, 각 경로의 결과를 집계하여 최종 답변을 도출하는 기법이다. 다수결(majority voting) 또는 가중 투표를 통해 더 견고하고 신뢰성 높은 답변을 얻을 수 있다.

## 왜 중요한가

[[LLM]]의 확률적 특성으로 인한 답변 불안정성을 보완하여 추론 정확도를 크게 향상시킬 수 있으며, 특히 수학, 논리 문제 같은 복잡한 추론 작업에서 실질적인 성능 개선을 제공한다.

## 관련 개념

- **상위 개념**: [[Prompting]](프롬팅)
- **하위 개념**: 
  - [[Chain-of-Thought]](사고의 연쇄): 단계별 추론 과정을 명시적으로 생성하도록 유도하는 기법
  - [[Majority-Voting]](다수결): 여러 샘플의 출력 중 가장 빈번한 답변을 선택하는 방식
- **연관 개념**:
  - [[Ensemble-Methods]](앙상블 방법): 여러 모델이나 샘플의 결과를 결합하는 일반적 기법
  - [[Temperature]](온도): 샘플링 다양성을 제어하여 다양한 추론 경로 생성에 영향을 주는 매개변수

## 비유로 이해하기

한 문제를 여러 학생에게 다르게 풀도록 한 후, 대다수가 같은 답을 제시하면 그것을 정답으로 채택하는 방식과 같다.

## 실제 사용 예시

수학 문제 "24를 2, 3, 4로 나눈 결과의 합은?"을 [[LLM]]에게 3번 반복 질문하여 [12, 13, 12]의 답변을 받으면, 다수결에 의해 최종 답은 12가 된다. [[Temperature]](온도)를 높여 다양한 추론 경로를 유도함으로써 각 시도마다 다른 계산 과정을 거치게 한다.
