# Vector-Database

## 정의
[[Vector-Database]](벡터 데이터베이스)는 [[Embedding]](임베딩) 형태의 고차원 벡터 데이터를 저장하고 검색하기 위해 최적화된 데이터베이스 시스템이다. 텍스트, 이미지, 음성 등 다양한 데이터를 [[Vector]](벡터)로 변환하여 의미론적 유사성(semantic similarity)을 기반으로 빠르게 검색할 수 있다.

## 왜 중요한가
[[LLM]](대형 언어 모델)과 [[AI-Terms/01-Terms/RAG]](검색 증강 생성) 기술의 대중화에 따라 대규모 임베딩 데이터를 효율적으로 관리하고 유사도 검색을 수행해야 하는 실무 수요가 급증했다. 전통 데이터베이스로는 고차원 벡터 데이터의 빠른 검색이 불가능하므로 필수적인 인프라 기술이 되었다.

## 관련 개념

**상위 개념:**
- [[Database]](데이터베이스)

**하위 개념:**
- [[Similarity-Search]](유사도 검색): 벡터 간의 거리나 유사도를 계산하여 가장 유사한 항목들을 반환하는 방식
- [[HNSW]](계층적 항해 가능 세계): 벡터 검색을 고속화하기 위한 근사 최근접 이웃(approximate nearest neighbor) 알고리즘

**연관 개념:**
- [[Embedding-Model]](임베딩 모델): 데이터를 벡터로 변환하는 모델
- [[Cosine-Similarity]](코사인 유사도): 벡터 간의 각도 기반 유사도를 측정하는 방식

## 비유로 이해하기
도서관에서 책의 내용이 비슷한 책들을 찾는 것과 같다. 전통 데이터베이스는 정확한 제목이나 저자명으로만 검색할 수 있지만, 벡터 데이터베이스는 책의 의미(주제, 감정 등)를 이해하여 주제가 비슷한 책들을 한 번에 찾아낼 수 있다.

## 실제 사용 예시
- ChatGPT의 플러그인이 사용자 문서를 [[Embedding]](임베딩)으로 변환하여 [[Vector-Database]](벡터 데이터베이스)에 저장한 후, 사용자 쿼리와 유사한 문서들을 검색해 [[LLM]](대형 언어 모델)의 입력으로 제공하는 [[AI-Terms/01-Terms/RAG]](검색 증강 생성) 시스템
- 전자상거래 플랫폼에서 상품의 이미지와 설명을 벡터화하여 "이 셔츠와 비슷한 상품 추천"이라는 유사 상품 검색 기능 구현
