# Embedding

## 정의
[[Embedding]](임베딩)은 단어, 문장, 이미지 등의 데이터를 고정된 차원의 숫자 벡터(numerical vector)로 변환하는 과정이다. 의미상 유사한 데이터들은 벡터 공간(vector space)에서 가깝게 배치되어 기계가 데이터의 의미를 이해하고 처리할 수 있게 한다.

## 왜 중요한가
[[Neural-Network]](신경망) 기반의 모든 모델이 텍스트와 이미지 같은 비정형 데이터를 처리하기 위해 임베딩을 필수적으로 사용하며, 임베딩의 질이 모델의 전체 성능을 좌우한다. 의미 기반 검색(semantic search)과 [[Recommendation-System]](추천 시스템)에서도 핵심 기술이다.

## 관련 개념
- **상위 개념**: [[Feature-Engineering]](피처 엔지니어링)
- **하위 개념**: 
  - [[Word2Vec]](워드투벡)
  - [[BERT]](버트)
- **연관 개념**: 
  - [[Vector-Database]](벡터 데이터베이스)
  - [[Similarity-Search]](유사도 검색)

## 비유로 이해하기
마치 사람들을 2D 지도 위의 좌표로 표시하는 것처럼, 단어나 문장 같은 의미를 가진 데이터를 숫자 좌표(벡터)로 변환한다. 성격이 비슷한 사람들이 지도상 가까이 위치하듯이, 의미가 유사한 데이터들은 벡터 공간에서도 가깝다.

## 실제 사용 예시
[[LLM]](대형 언어 모델)이 "왕 - 남자 + 여자 ≈ 왕비"라는 의미 관계를 계산할 수 있는 것은 각 단어가 임베딩 벡터로 표현되어 있기 때문이다. [[AI-Terms/01-Terms/RAG]](검색 증강 생성) 시스템에서 사용자 질문과 문서를 같은 벡터 공간에 임베딩한 후 유사도를 비교하여 관련 문서를 찾는다.
