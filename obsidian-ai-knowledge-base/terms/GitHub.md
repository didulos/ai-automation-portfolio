# GitHub

## 정의
[[GitHub]](깃허브)는 [[Git]](깃) 기반의 분산 버전 관리 시스템을 웹 기반으로 제공하는 플랫폼이다. 개발자들이 코드를 저장, 공유, 협업하고 프로젝트를 관리할 수 있는 중앙 저장소 역할을 한다.

## 왜 중요한가
AI/ML 프로젝트의 코드 버전 관리, 팀 협업, 코드 리뷰(code review)를 효율적으로 처리할 수 있으며, 오픈소스 생태계의 중심이므로 최신 머신러닝 모델과 라이브러리에 접근하는 데 필수적이다.

## 관련 개념
- **상위 개념**: [[Version-Control]](버전 관리)
- **하위 개념**: 
  - [[Pull-Request]](풀 리퀘스트)
  - [[Repository]](저장소)
- **연관 개념**: 
  - [[CI/CD]](지속적 통합/배포)
  - [[Git]](깃)

## 비유로 이해하기
GitHub는 여러 명이 함께 책을 집필할 때, 각자의 수정 사항을 기록하고 충돌을 해결하며 최종 버전을 관리하는 "편집장"과 같다.

## 실제 사용 예시
```
팀원이 새로운 특성 엔지니어링(feature engineering) 코드를 작성한 후 
Pull Request를 생성하면, 다른 개발자들이 코드를 검토하고 병합(merge)한다.
```
