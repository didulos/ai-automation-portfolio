# Version-Control

## 정의
소프트웨어 코드나 문서의 변경 이력을 추적하고 관리하는 시스템으로, 여러 개발자가 동시에 작업할 때 변경사항을 기록하고 이전 버전으로 되돌릴 수 있게 한다. [[VCS]](버전 관리 시스템)는 누가, 언제, 무엇을, 왜 변경했는지를 명확히 기록함으로써 코드의 무결성을 보장한다.

## 왜 중요한가
실무에서 팀 협업 시 코드 충돌(conflict)을 체계적으로 관리하고, 문제 발생 시 특정 버전으로 빠르게 복구할 수 있어 개발 효율성과 안정성을 크게 향상시킨다. 또한 코드 리뷰와 감시 추적(audit trail)을 가능하게 해 소프트웨어 품질 관리의 핵심 도구다.

## 관련 개념
- **상위 개념**: [[Software-Development-Lifecycle]](소프트웨어 개발 생명주기)

- **하위 개념**:
  - [[Git]](깃): 분산 버전 관리 시스템으로 가장 널리 사용되는 [[VCS]]
  - [[Commit]](커밋): 코드 변경사항을 저장소에 기록하는 단위 작업

- **연관 개념**:
  - [[CI/CD]](지속적 통합/배포): 버전 관리와 함께 작동하는 자동화 파이프라인
  - [[Code-Review]](코드 리뷰): [[Version-Control]] 시스템 위에서 수행되는 협업 프로세스

## 비유로 이해하기
구글 문서(Google Docs)의 "버전 기록" 기능처럼, 문서의 모든 수정 내역을 시간순으로 저장했다가 원하는 시점의 버전으로 언제든지 돌아갈 수 있는 것과 같다.

## 실제 사용 예시
```
git commit -m "머신러닝 모델 성능 개선"
# 1주일 뒤 버그 발견 시
git revert <commit-hash>  # 이전 버전으로 복구
```
