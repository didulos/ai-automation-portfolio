# API

## 정의
[[API]](응용 프로그래밍 인터페이스)는 소프트웨어 간 통신을 위한 규칙과 프로토콜의 집합으로, 한 프로그램이 다른 프로그램의 기능이나 데이터를 요청하고 받을 수 있도록 정의된 인터페이스다. [[REST]], [[GraphQL]] 등 다양한 아키텍처 방식으로 구현될 수 있다.

## 왜 중요한가
현대의 AI/ML 시스템은 모델 서빙(model serving)부터 데이터 수집, 외부 서비스 연동까지 API를 통해 작동하므로, API 설계와 활용 능력은 실무에서 필수적이다. 특히 [[LLM]] 기반 애플리케이션(예: [[AI-Terms/01-Terms/RAG]])에서 외부 데이터소스나 도구와의 통신을 위해 API가 핵심 역할을 한다.

## 비유로 이해하기
API는 레스토랑의 메뉴판과 같다. 손님(프로그램)이 메뉴판(API)에 명시된 대로 주문(요청)하면, 주방(다른 프로그램)이 음식(응답)을 준비해 제공한다. 손님은 주방의 내부 작동 방식을 몰라도 메뉴판을 통해 원하는 것을 얻을 수 있다.

## 실제 사용 예시
```
OpenAI API를 호출하여 LLM에 프롬프트를 전송하고 생성된 텍스트를 받는다:
response = client.chat.completions.create(model="gpt-4", messages=[...])
```

데이터 파이프라인에서 [[ETL]](추출-변환-로드) 프로세스 중 외부 데이터소스의 API를 통해 실시간 데이터를 수집한다.
