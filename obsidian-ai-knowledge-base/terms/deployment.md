# deployment

## 정의

학습이 완료된 [[AI-Terms/01-Terms/Machine-Learning]](머신러닝) 모델을 프로덕션(production) 환경에 배포하여 실제 사용자의 데이터에 대해 예측(prediction)을 수행하는 프로세스이다. 모델이 개발 단계에서 벗어나 실제 비즈니스 문제를 해결하도록 운영되는 단계를 의미한다.

## 왜 중요한가

아무리 좋은 모델도 배포되지 않으면 비즈니스 가치를 창출할 수 없으며, 배포 후 모델의 성능 모니터링(monitoring)과 유지보수(maintenance)는 장기적인 안정성과 정확도 유지에 필수적이다.

## 관련 개념

- **상위 개념**: [[AI-Terms/01-Terms/MLOps]](머신러닝 운영)
- **하위 개념**: 
  - [[API]](API 배포) - REST API 형태로 모델을 서빙
  - [[Batch-Processing]](배치 처리) - 대량의 데이터를 주기적으로 처리
- **연관 개념**: 
  - [[Model-Monitoring]](모델 모니터링) - 배포 후 성능 추적
  - [[Model-Versioning]](모델 버전 관리) - 배포된 모델의 버전 관리

## 비유로 이해하기

완성된 자동차를 공장에서 생산 라인으로 옮기고, 실제 도로에서 고객들이 운전할 수 있도록 하는 것과 같다. 설계 단계(개발)가 끝나면 실제 사용 가능한 형태로 만들어야(배포) 의미가 있다.

## 실제 사용 예시

데이터 과학팀이 고객 이탈 예측 모델(churn prediction model)을 개발한 후, 이를 마케팅 팀이 사용할 수 있도록 클라우드 서버에 API로 배포했다. 매월 모델의 정확도를 모니터링하며 필요시 재훈련된 새 모델로 업데이트한다.
