# PEP 668

## 정의
[[PEP-668]](PEP 668)은 Python 패키지 관리자(package manager)의 보안과 시스템 무결성을 보호하기 위해 도입된 Python Enhancement Proposal으로, 시스템 Python 환경에 직접 패키지를 설치하는 것을 제한하는 정책이다. 주로 데비안(Debian)과 우분투(Ubuntu) 같은 리눅스 배포판에서 시스템 패키지 매니저와 [[pip]](pip)의 충돌을 방지하기 위해 적용된다.

## 왜 중요한가
개발자들이 시스템 Python에 직접 패키지를 설치하려 할 때 permission 오류를 만나게 되며, [[가상환경]](Virtual-Environment) 사용을 강제함으로써 시스템 안정성과 프로젝트 격리를 보장한다.

## 비유로 이해하기
공동 냉장고(시스템 Python)에 아무나 음식을 가져다놓으면 다른 사람 음식이 상할 수 있으니, 개인용 도시락(가상환경)을 준비해서 그곳에만 음식을 담도록 강제하는 것과 같다.

## 실제 사용 예시
```bash
# PEP 668으로 인한 오류 발생
pip install numpy
# error: externally-managed-environment

# 해결: 가상환경 사용
python -m venv myenv
source myenv/bin/activate
pip install numpy  # 정상 작동
```
