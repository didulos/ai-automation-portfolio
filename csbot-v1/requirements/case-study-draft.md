## 문제

스마트스토어를 혼자 운영하면 하루가 CS로 시작해서 CS로 끝납니다.

"배송 언제 와요?", "교환 어떻게 해요?", "반품하면 배송비 누가 내요?"

같은 질문이 하루에 수십 번 반복됩니다. 답변은 항상 똑같은데, 직접 타이핑하는 시간은 쌓여만 갑니다. 주문 처리, 상품 촬영, 마케팅에 써야 할 시간이 CS에 잠식되고 있었습니다.

**실제 상황:**
- 반복 문의 비율: 전체 CS의 70~80% 추정 (배송·교환·반품·취소 4개 카테고리 집중)
- 셀러 대응 시간: 문의당 평균 3~5분
- 운영 시간 외 문의: 답변 지연 → 고객 불만

---

## 솔루션

**n8n + Claude API + Airtable로 만든 CS 자동화 봇**

코딩 없이 5분 온보딩으로 시작하고, 셀러가 직접 FAQ를 관리할 수 있는 구조로 설계했습니다.

### 아키텍처

```
고객 문의 (카카오톡 채널 / 웹 채팅 위젯)
    ↓
n8n Webhook 수신
    ↓
Airtable에서 스토어 정보 + FAQ 조회
    ↓
Claude API로 맞춤 답변 생성
    ↓
고객에게 응답 반환 + 대화 이력 저장
    ↓
(폴백 발생 시) 셀러에게 이메일 알림
```

### 핵심 설계 원칙

**1. 스토어 정보 자동 치환**
FAQ 응답 템플릿에 변수(`{{배송사}}`, `{{교환기간}}`)를 심어 두고, Claude가 해당 스토어 정보로 치환해서 답변합니다. 셀러마다 다른 정책이 그대로 반영됩니다.

**2. 폴백 우선 설계**
봇이 답하기 어려운 문의는 "담당자에게 전달하겠습니다"로 응답하고, 셀러에게 즉시 알림을 보냅니다. 봇이 모르는 척하지 않습니다.

**3. 5분 온보딩**
셀러는 폼 하나만 제출하면 됩니다. 스토어명·배송사·교환 정책 입력 → Airtable에 자동 저장 → FAQ 10개 자동 생성 → 봇 즉시 활성화.

**4. 채널 듀얼 지원**
카카오톡 채널 챗봇과 스마트스토어 임베드 웹 위젯을 동일한 백엔드로 처리합니다.

### 스택

| 역할 | 도구 |
|------|------|
| 워크플로우 엔진 | n8n (셀프호스팅) |
| AI 응답 생성 | Claude API (claude-sonnet-4-6) |
| 데이터 저장 | Airtable |
| 채널 1 | 카카오 i오픈빌더 스킬 서버 |
| 채널 2 | 웹 채팅 위젯 (스마트스토어 임베드) |

---

## 결과

*테스트 환경: 15개 시나리오 (FAQ 10개 + 엣지케이스 5개)*

| 지표 | 결과 |
|------|------|
| **FAQ 자동 응답률** | **80% (12/15)** |
| 평균 응답 시간 | 3초 내외 |
| 폴백(모름) 비율 | 20% 이하 |
| 온보딩 소요 시간 | 5분 이내 |
| 인프라 비용 (Claude API) | 일 540건 기준 약 5,000원 |

**잘 처리된 유형:** 배송 소요일, 교환 조건, 반품·환불, 주문 취소, 재입고, 영업시간, 결제 수단, 제주·도서산간 배송

**폴백이 발생한 유형:** 특정 성분·소재 문의처럼 상품별 맞춤 정보가 필요한 케이스 → FAQ 추가로 해결 가능
