## 데모 개요

| 항목 | 내용 |
|------|------|
| **대상** | 스마트스토어 운영 셀러 (잠재 고객) |
| **소요 시간** | 약 10~15분 |
| **준비물** | 웹 채팅 위젯 열어 둔 브라우저, Airtable Conversations 탭 |
| **핵심 메시지** | "반복 CS 문의를 자동으로 처리하고, 셀러에게는 꼭 필요한 문의만 알림으로 전달합니다" |

## 시나리오 A — 기본 CS 응답

> **목적:** 고객 문의 → 즉시 자동 응답 + 대화 문맥 유지 확인

### 도입 멘트

> "스마트스토어 운영하시면 배송 문의, 교환/반품 문의가 하루에 몇 번씩 반복되죠?
> 그 문의들을 봇이 대신 처리하면 어떨까요. 지금 바로 시연해 드릴게요."

### 실행 단계

| 단계 | 입력 | 확인 포인트 |
|------|------|-----------|
| 1 | 채팅창 열기 | 웹 채팅 위젯 정상 로드 확인 |
| 2 | **"주문한 지 3일 됐는데 배송이 아직인데요"** 입력 | 배송사·출고일·당일 마감시간 포함한 답변 출력 (~3초) |
| 3 | **"그럼 교환은 어떻게 해요?"** 입력 | "그럼"으로 시작하는 문맥 연결 → 교환 기간·조건 포함 답변 출력 |
| 4 | Airtable Conversations 탭 열기 | 두 대화가 실시간 기록된 것 확인 |

### 강조 포인트

- "그럼"이라는 연결 표현에도 봇이 이전 대화 맥락을 이해하고 교환 정책으로 자연스럽게 연결
- 스토어명·배송사·교환 기간이 **이 가게 정보**로 실제 치환되어 있음
- 셀러가 아무것도 하지 않아도 대화 이력이 Airtable에 자동 저장됨

## 시나리오 B — 봇이 모르는 질문 처리

> **목적:** 봇이 모르는 질문은 스스로 인식하고 셀러에게 넘기는 흐름 시연

### 도입 멘트

> "물론 봇이 모든 걸 다 알 순 없죠. 답하기 어려운 문의가 오면 어떻게 되는지 보여 드릴게요."

### 실행 단계

| 단계 | 입력 | 확인 포인트 |
|------|------|-----------|
| 1 | **"이 제품 OO 성분 들어있나요?"** 입력 | 봇 → "담당자에게 전달하겠습니다" 폴백 응답 출력 |
| 2 | Airtable Conversations 탭 확인 | WasFallback = true 로 기록된 것 확인 |
| 3 | 이메일 또는 Telegram 앱 확인 | "폴백 발생: [고객 메시지]" 알림 수신 확인 |

### 강조 포인트

- 봇이 무조건 답하지 않음 — **모르면 솔직하게** 담당자 연결
- 셀러는 알림으로 즉시 파악 가능
- FAQ에 성분 정보를 추가하면 다음부터는 자동 답변 — **개선이 쉬움**

## 시나리오 C — 5분 온보딩

> **목적:** 내 가게 정보 입력 → 즉시 봇 활성화까지의 전체 흐름 시연

### 도입 멘트

> "그럼 이 봇을 내 스토어에 붙이려면 뭘 해야 하냐고요? 폼 하나만 채우시면 돼요."

### 실행 단계

| 단계 | 액션 | 확인 포인트 |
|------|------|-----------|
| 1 | 온보딩 폼 URL 열기 (`/form/onboarding`) | 스토어명·배송사·교환기간 등 입력 폼 표시 |
| 2 | 폼 항목 직접 채워 가며 Submit | 완료 페이지로 리다이렉트 (ArumDri 브랜드 완료 화면) |
| 3 | Airtable Stores 탭 열기 | 방금 입력한 가게 정보가 새 레코드로 생성된 것 확인 |
| 4 | 새 레코드의 `StoreID` 숫자 확인 | (예: 2) |
| 5 | `chat-widget.html` → `CONFIG.STORE_ID` 를 확인된 숫자로 변경 후 새로고침 | 위젯 헤더에 새 스토어명 표시 |
| 6 | 채팅창에 **"배송 얼마나 걸려요?"** 입력 | 방금 입력한 가게 정보(배송사·출고일)로 치환된 답변 출력 |

> **주의:** 단계 5의 STORE_ID 변경은 데모 전에 미리 해 두거나, 데모 중 Airtable에서 Auto Number 확인 후 라이브로 수정해도 됨.
> 기본값(`STORE_ID: '1'`)은 ArumDri 샘플 스토어이므로 새 스토어로 테스트하려면 변경 필요.

### 강조 포인트

- 설정 난이도: 폼 1개 제출 (코딩 불필요)
- FAQ는 Airtable에서 텍스트만 수정 — 비개발자도 운영 가능
- 웹 위젯은 스크립트 한 줄 붙여넣기로 스토어 페이지에 삽입

## 마무리 멘트 & Q&A 포인트

> "정리하면: 반복 CS는 봇이 처리하고, 봇이 모르는 건 셀러에게 알림이 옵니다.
> 가게 정보 설정은 폼 하나, FAQ 수정은 Airtable 텍스트 수정으로 끝납니다."

### 자주 나오는 질문

| 질문 | 답변 |
|------|------|
| 카카오톡도 되나요? | 카카오 채널에 스킬 서버로 연결하면 동일하게 동작합니다 |
| 비용이 얼마예요? | Claude API 기준 하루 540건까지 5,000원 이내 |
| 직접 운영해야 하나요? | n8n 셀프호스팅 또는 호스팅 서비스 선택 가능 |
| 대화 이력은 얼마나 보관돼요? | Airtable 무료 티어 기준 1,000건 — 유료 전환 시 무제한 |
| 답변이 틀리면요? | FAQ 텍스트를 수정하면 즉시 반영됩니다. 이력도 남아서 어떤 문의가 오는지 파악 가능합니다 |
